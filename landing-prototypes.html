Design notes
Provider A — Dashboard welcome
Leads with a personal greeting and headline stats. Four action cards give quick access to the most common tasks. Recent activity feed keeps providers engaged. Best for providers who log in regularly to manage their listings.
Provider B — Growth focus
Opens with a "reach" headline to motivate providers (X parents saw your listings). Surfaces a contextual tip to improve a specific listing. Compact performance table replaces individual listing cards. Better for first-time or infrequent providers who need the "why".
Parent A — Discovery hub
Closest to the current browse page but enriched with a search bar, category filters, and a value proposition banner. Keeps the focus on exploring. Best for new or casual users who don't yet have children set up or saved activities.
Parent B — Personalised home
Uses the kids data already in the platform to create a personalised experience. Each child gets their own section with their saved activities. Surfaces pending trials prominently. Requires kids to be set up to be useful — fallback needed for new users with no kids yet.
